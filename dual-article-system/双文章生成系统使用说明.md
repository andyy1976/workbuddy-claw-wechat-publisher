# 📝 智能双文章生成系统使用说明

## 🎯 系统概述

**智能双文章生成系统** 是一个自动化内容创作工具，每次执行时：

1. **智能选择主题** - 从预设的热门主题库中选择最热门的主题
2. **双文章生成** - 同时生成小红书风格和公众号风格两篇文章
3. **自动发布** - 将两篇文章都发布到微信公众号草稿箱
4. **智能备份** - 自动保存生成的文章和发布记录

## ✨ 核心功能

### 🔥 智能选题
- 基于热度评分算法选择最热门主题
- 主题库包含5个高热度领域
- 每个主题都有完整的关键词、标签和emoji配置

### 📱 双风格文章生成
- **小红书风格**：亲切、实用、有故事感（1000字以内）
- **公众号风格**：专业、深度、结构化
- 两篇文章都围绕同一个主题，但风格和受众定位不同

### 🚀 自动发布
- 一键发布两篇文章到微信公众号草稿箱
- 支持封面图配置
- 完整的发布状态监控和日志记录

### 💾 智能管理
- 生成的文章自动保存到本地
- 完整的执行日志和摘要报告
- 支持历史文章查询和管理

## 🛠️ 快速开始

### 步骤1：环境准备
确保系统已安装：
- Node.js v14.0.0 或更高版本
- 基本的npm包管理器

### 步骤2：配置微信公众号API
编辑 `automation_config.json` 文件：

```json
{
  "appId": "你的微信公众号AppID",
  "appSecret": "你的微信公众号AppSecret",
  "thumbMediaId": "封面图素材ID"
}
```

或者设置环境变量：
```bash
export WECHAT_APP_ID="your_app_id"
export WECHAT_APP_SECRET="your_app_secret"
export WECHAT_THUMB_MEDIA_ID="your_thumb_media_id"
```

### 步骤3：运行系统测试
```bash
node test_dual_article.js
```

### 步骤4：执行双文章生成
```bash
node dual_article_generator.js
```

## 🔧 配置详解

### 主题库配置
系统内置5个热门主题，每个主题包含：
- **id**: 唯一标识符
- **title**: 主题标题
- **category**: 分类
- **keywords**: 关键词列表
- **trendScore**: 热度评分（0-100）
- **xiaohongshuTags**: 小红书标签
- **emoji**: 推荐使用的emoji

### 文章风格配置

#### 小红书风格特点：
- 亲切的朋友聊天语气
- 实用性强，提供具体建议
- 包含emoji和表情符号
- 结尾有相关话题标签
- 字数控制在1000字以内

#### 公众号风格特点：
- 专业的行业分析视角
- 结构化的内容组织
- 数据驱动的观点论证
- 全面的趋势预测
- 标准的公众号文章格式

## 🔍 故障排除

### 常见问题1：微信API配置错误
**症状**：发布失败，提示"invalid credential"或"access_token"
**解决**：
1. 检查 `automation_config.json` 中的AppID和AppSecret
2. 确认微信公众号已开通相关API权限
3. 尝试重新获取Access Token

### 常见问题2：文章生成失败
**症状**：文章内容为空或格式错误
**解决**：
1. 运行 `node test_dual_article.js` 检查系统环境
2. 确认Node.js版本符合要求
3. 检查是否有足够的磁盘空间

### 常见问题3：文件保存失败
**症状**：提示"无法写入文件"或"权限不足"
**解决**：
1. 检查当前用户的文件写入权限
2. 确认 `generated_articles` 目录存在且有写入权限
3. 检查磁盘空间是否充足

### 集成到现有系统
```javascript
const DualArticleGenerator = require('./dual_article_generator.js');

async function runCustomized() {
    const generator = new DualArticleGenerator();
    
    // 自定义主题选择逻辑
    const customTopic = generator.hotTopics.find(t => t.id === 'web3_evolution');
    
    // 只生成不发布
    const xhsArticle = generator.generateXiaohongshuArticle(customTopic);
    const wechatArticle = generator.generateWechatArticle(customTopic);
    
    // 自定义处理逻辑
    console.log('生成完成，文章长度：', {
        xhs: xhsArticle.length,
        wechat: wechatArticle.length
    });
}

runCustomized();
```

        xiaohongshuTags: ['#标签1', '#标签2'],
