## 缩略图生成流程 (每次运行都会重新生成!)

**完整流程** (`enhanced-engine.js` 第580-620行附近):
1. `matchThumbnail(title, digest)` (`article-enhancer.js`)
   - 调用 `generateImageKeywords()` → AI 生成英文关键词
   - 调用 `searchStockImage(keyword, { source: 'pexels' })` → 从 **Pexels API** 搜索
   - 如果 Pexels 失败 → 降级到 **Pixabay API**
   - 返回 `images[0]` (`{ url, source, photographer }`)

2. `downloadImage(thumbnail.url, thumbnailPath)`
   - 下载图片到本地 `output/thumb_xxx.jpg`

3. `uploadWechatImage(thumbnailPath, 'thumb')`
   - 上传到微信素材库
   - **返回新的 `media_id`** (每次都不同!)

4. `publishWechatWithThumb(title, digest, contentHtml, thumbMediaId)`
   - 用**新的 `media_id`** 发布到微信草稿箱

**降级方案** (如果上述任何一步失败):
- 使用 `config/user-config.json` 里的 `wechat.thumbMediaId` (默认缩略图)
- ⚠️ **这个默认 `thumbMediaId` 可能已过期!**

## 正确运行方式

### ✅ 正确: `run-engine.bat` 或 `go.bat`
```bash
# 方式1: 双击 run-engine.bat
# 方式2: 命令行
cd D:\scsaicms\workbuddy-claw-wechat-publisher
node scripts\enhanced-engine.js
```

**执行流程** (`enhanced-engine.js` 的 `main()` 函数):
1. `selectBestTopic()` - RAG 话题选择
2. `generateTitle(topic)` - AI 生成标题
3. `writeArticle(topic)` - AI 深度写作
4. `checkContent(title, article)` - 内容安全检查
5. `mdToHtml(content)` - Markdown → 微信 HTML
6. `matchThumbnail(title, digest)` - 从 Pexels/Pixabay 搜索缩略图
7. `uploadWechatImage(thumbnailPath, 'thumb')` - **上传缩略图到微信** (获取 `media_id`)
8. `publishWechatWithThumb(title, digest, contentHtml, thumbMediaId)` - 发布到微信草稿箱
9. `saveArticle(title, content, html, topic)` - 写入 CMS 数据库

### ❌ 错误: 调试 `server/routes/publish.js` 来解决微信发布问题
- 这是 HTTP API，给 Web UI 调用的
- **不是** 命令行直接运行的引擎
- 如果你在调试"微信发布失败"，看这里只会浪费时间
